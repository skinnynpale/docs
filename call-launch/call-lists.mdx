В разделе «Колл-листы» вы можете создавать, запускать, приостанавливать и форматировать списки контактов, которые Саша получает в работу.

Система последовательно обзванивает их в соответствии с установленными настройками.

## Ручная загрузка контактов

Для ручного добавления контактов в список обзвона:

1. Перейдите в раздел «Колл-листы»
2. Выберите нужный колл-лист
3. Перейдите в подраздел «Управление»
4. Нажмите на кнопку «Загрузить контакты»
5. Перетащите предварительно форматированную базу контактов со своего компьютера

### Шаблон базы контактов

**Из важного:**

Таблица должна содержать как минимум два столбца:
- `phone` — номер телефона в любом формате
- `tags` — опциональные теги контактов через запятую

Вы можете обогатить базу контактов переменными, которые вы ранее указали в «Скиллбейсе». Так Саша будет обладать специфической информацией о каждом номере телефона при обзвоне. Подробнее об этом читайте в разделе «Переменные».

Подробнее о столбце `tags` и зачем он нужен читайте в разделе «Когортный анализ».

### Предпросмотр загрузки

После загрузки перед вами появится окно предпросмотра загруженных контактов.

**Перепроверьте корректность загрузки.**

<Danger>
Обратите внимание: если в поле «дополнительная информация» на месте переменных стоят пустые символы (знаки вопроса) — значит система не смогла прочитать переменные.

Зачастую, проблема кроется в кодировке исходного файла — повторно изучите раздел «Кодировка базы контактов».
</Danger>

Если загрузка произошла корректно — нажмите на кнопку «Загрузить контакты».

### Отчет о загрузке

Перед вами появится отчет о загрузке контактов. Здесь вы можете отследить:
- Добавленные в колл-лист контакты
- Контакты, которые система не смогла пропустить

Подробнее о причинах пропуска контактов читайте в разделе «Ограничения на работу с колл-листами».

## Автоматизированная загрузка контактов

Для того, чтобы каждый раз не загружать контакты вручную вы можете настроить входящую интеграцию на прием контактов.

### Настройка:

1. Перейдите в раздел «Колл-листы»
2. Выберите нужный колл-лист
3. Перейдите в подраздел «Интеграции»
4. Нажмите на кнопку «Создать интеграцию»
5. Введите название интеграции, ее описание
6. В выпадающем списке «Тип интеграции» установите значение «API (Webhook)»
7. Нажмите на кнопку «Создать»

Перед вами откроется всплывающее окно с URL адресом созданного вебхука.

### Правила использования вебхука:

- **Никому не показывайте созданный URL**
- Отправляйте POST-запросы с данными контактов на этот адрес, включая возможные переменные и их значения
- **Формат данных:** JSON с массивом контактов

**Пример:**
```json
[{"phone": "79000000000", "tags": ["tag1", "tag2"]}]
```

## Загрузка переменных

С помощью раздела «Как обогатить контекст Саши о каждом номере телефона в отдельности?» вы можете написать инструкции для Саши таким образом, чтобы перед звонком она собирала предварительный контекст о клиенте и вела переговоры с учетом него.

### Три шага для работы с переменными:

#### 1. Определить информацию о клиенте

Понять какую информацию о клиенте мы хотим использовать в ходе диалога.

#### 2. Присвоить переменные

Присвоить каждому типу информации свою переменную в формате `{variable}`.

**Пример:**
```
{client_name} // имя клиента
{client_business} // сфера деятельности клиента
{client_surname} // отчество клиента
{client_history} // история сделок клиента
```

#### 3. Разнести по скиллбейсу

Разнести переменные по подходящим по смыслу частям скиллбейса.

**Например:**

В аргументах и фактах:
```
*Информация о клиенте*
Ученик скорее всего учится в {client_school}
```

В готовых ответах:
```
Даа, {client_name} добрый день… . подскажите, вы ведь по стройматериалам еще работаете, верно?
```

### Подтягивание переменных при загрузке

Для того чтобы наши труды не пропали даром нам нужно подтянуть переменные на этапе загрузки контактов.

Для этого перед загрузкой базы внесите в таблицу с контактами дополнительные столбцы в соответствии с теми переменными, которые вы указали в скиллбейсе.

В каждой строчке получившейся таблицы должен лежать номер телефона, теги и переменные принадлежащие ему.

### Шаблон базы с переменной {client_name}

<Note>
Обратите внимание, что в скиллбейсе переменные указываются в фигурных скобках, в то время как в таблице названия столбцов указываются без них.
</Note>

<Warning>
Имейте ввиду, что при активации колл-листа, в котором выбран скиллбейс у которого есть переменные, а в самом колл-листе их не будет — система проставит на месте переменных в контекстном окне нейросети пустые пробелы.
</Warning>

## Ограничения на работу с колл-листами

Существует несколько ограничений по работе с колл-листами. Это системные ограничения, которые внедрены чтобы исключить вероятность инцидентов по причине человеческого фактора.

### 1. 1 Колл-лист = 1 скиллбейс

Вся система построена по логике **1 скиллбейс = 1 колл-лист**. Вы можете загружать разные базы в один колл-лист, таким образом упорядочив свою работу.

### 2. Автоматический пропуск дублей

Во-вторых, вы не можете загрузить два одинаковых контакта в один колл-лист. 

Система автоматически обнаружит дубли номеров телефонов и поместит в колл-лист один единственный контакт. Таким образом, мы исключаем шанс повторного звонка с тем же скриптом одному абоненту.

### 3. Двухнедельный отпуск

В-третьих, вы не можете ставить в очередь на прозвон контакт более **1 раза за две недели**. 

Это ограничение действует на уровне всей организации. Система не пропустит контакт в колл-лист, если он был обработан в рамках какого-либо колл-листа за последние 2 недели.

По прошествию двух недель вы сможете еще раз позвонить этому абоненту. Для этого перейдите в колл-лист, куда он был загружен изначально, перейдите в подраздел «Контакты» и удалите этот номер из колл-листа.

## Удаление контактов

Вы можете вручную удалить контакты из колл-листа.

### Как удалить контакты:

1. Перейдите в раздел «Колл-листы»
2. Выберите нужный колл-лист
3. Перейдите в подраздел «Контакты»

Перед вами откроется таблица с загруженными в этот колл-лист контактами.

По каждому контакту вы видите:
- Номер телефона
- Дату добавления
- Теги
- Переменные
- Статус обработки

### Фильтрация и удаление

Сверху есть фильтр по статусу обработки. Используйте его, чтобы выбрать только обработанные и необработанные контакты и удалить их.

Для удаления контактов:
1. Отфильтруйте таблицу или выберите нужные номера
2. Нажмите на квадрат в левом верхнем углу таблицы
3. Нажмите на кнопку «Удалить все контакты»
